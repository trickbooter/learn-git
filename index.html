Repository

  • A persistant store for your source code
  • Records change history over time

Repository Areas

  • Working Copy - where you do the work
  • Staging/Stage/Index - a commit shopping basket used to organise what goes into a commit
  • Repo - the immutable commit store. The latest commit is referenced as HEAD

Branches

  • ISOLATE your changes from everyone else
  • Make all the mistakes you like on a private branch (but try not to)
  • Switch to other branches to make changes (i.e. make hotfix for release)

The Repo Object Model - DAG

The Repository is a Directed Acyclic Graph

  • Each commit is uniquely identified (circles)
  • Each commit points to only one parent commit (black arrows)
  • Pointers are acyclic (linear, can't create a closed loop of commits)

Refs

  • A reference (or ref) is a pointer to a commit or another ref
  • They come in the following flavours: local branch, remote branch and tag
  • A branch is identified by a ref pointing to the last (HEAD) commit on that branch

## Branching Objective: Learn how to create a branch Branching isolates your development effort from other branches.

## Branching Using Git Bash ``` git branch my-feature-branch-a ``` Branches the current branch's latest commit to my-feature-branch-a.
## Branching Review Look at the refs... ![](https://raw.githubusercontent.com/trickbooter/learn-git/gh-pages/assets/branches-ui.PNG) Both master and feature-branch-a point to the same commit. A branch is just a reference to a commit, remember our DAG. The branch doesn't physically create a new version of files like it would on TFS or SVN, it just adds a new ref. Since we haven't made any changes, both master and feature-branch-a point to the same commit. This is why git branching is so fast. ![](https://raw.githubusercontent.com/trickbooter/learn-git/gh-pages/assets/refs-small.png)
## Switch Between Branches Objective: Learn how to switch between branches
## Switch Between Branches Using SourceTree * Right click the branch that you wish to checkout * Click checkout * The branch in *bold* is the checked out branch ![](https://raw.githubusercontent.com/trickbooter/learn-git/gh-pages/assets/branches-ui.PNG) this will swap the files around in your working folder
## Switch Between Branches Using Git Bash ``` git checkout feature-branch-a ``` You can combine a branch and switch in one command using ``` git checkout -b feature-branch-a ```
## Merging (Fast-Forward) Objective: Learn how to merge. We will make a change on feature-branch-a then merge feature-branch-a back into master. ### What's a Merge and what are we doing? Merging allows you to update a branch with the contents of another. Such as merging your-amazing-feature into the master branch. We will focus on an easy merge scenarios at this stage. This is a special case called a *fast-forward* merge. ### fast forward merges Remember that a branch is just a *ref* to a commit. If the following is true * commits to feature-branch-a * no commits to master when we merge feature-branch-a into master they will be identical, so a fast-forward merge _moves the master pointer to the feature-branch-a HEAD_
## Make a change on feature-branch-a Merges target the current branch. If we are on master and we merge feature-branch-a, we are asking for feature-branch-a to be merged into master . * Switch to feature-branch-a * Make a change to c:\git\learn-git\README.txt & commit ![](https://raw.githubusercontent.com/trickbooter/learn-git/gh-pages/assets/feat-branch-a-change-small.PNG) note that master still points to the previous commit, and feature-branch-a has our latest commit
## Merge using Source Tree * Switch to master (our merge target branch) * Right click the latest feature-branch-a commit and choose merge ![](https://raw.githubusercontent.com/trickbooter/learn-git/gh-pages/assets/merge-ui.png) As no other changes had occured on master, a merge will result in feature-branch-a and master becoming identical. Git recognises this and simply moves the master ref to the same commit as feature-branch-a.
## Merge using Git Bash ``` git checkout feature-branch-a ``` * Make a change to README.md & commit ``` git commit -m "changed README.md" ``` * Switch to master and merge ``` git checkout master git merge feature-branch-a ```

Git Remotes, Branching & Merging

## Remotes Objective: Learn what a remote is. A remote is another repository that you can link to your repository. **That's all**. There is nothing special about a remote, it is just another repository. You can mark one remote as _default_, and convention suggests we name this _origin_. Typically our default remote is BitBucket or GitHub.
